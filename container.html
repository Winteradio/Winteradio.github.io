02

Two Real Bugs

#25 Use-After-Free in DynamicArray Move Constructor closed
The move constructor moved m_data but not m_allocator — the moved-from object kept a live, default-constructed allocator with no real authority over the transferred memory, so its destructor freed memory it no longer owned. Fix: move the allocator alongside the data in the initializer list, same fix ported to move-assignment.
#24 HashMap::operator[] Reference Invalidation during Rehash open
m_map[A] = m_map[B] can crash: the RHS reference is taken first, then the LHS access triggers a rehash that relocates and frees the old backing array — the assignment then reads through a dangling reference. Doesn't happen on std::unordered_map (node-based, stable addresses); does happen here because contiguous open addressing is exactly what makes lookups cache-friendly. Logged as an inherent trade-off of the design, not patched over.
