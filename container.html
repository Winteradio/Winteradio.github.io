02

Two Real Bugs

#25 Use-After-Free in DynamicArray Move Constructor closed
Move ctor moved m_data but not the allocator — the moved-from object's destructor freed memory it no longer owned. Fixed by moving the allocator too.
#24 HashMap::operator[] Reference Invalidation during Rehash open
m_map[A] = m_map[B] can crash: taking the RHS reference, then the LHS access triggers a rehash that frees the old array — an inherent trade-off of contiguous open addressing, not patched over.
