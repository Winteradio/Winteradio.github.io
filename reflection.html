C++17 Reflection

GENERATE / PROPERTY / METHOD — modeled after Unreal's UCLASS/UPROPERTY/UFUNCTION. Register type, members, and methods at compile time via C++17 inline static, header-only, no linker errors.

Type / Property / Method

single inheritance only — that's what makes Cast<T> safe

TypeInfo models one SuperType per class, not a set of bases. Cast<T> walks that single chain — one unambiguous path up, so a cast is a walk, not a search across a DAG, and dynamic_cast is never needed.

type map — registration
TypeInfo  «single SuperType chain»
  GetProperty(name)  -> PropertyInfo*
  GetMethod(name)    -> MethodInfo*

PropertyInfo
  Get(instance)          -> const void*
  Set(instance, value)
  └─ ContainerPropertyInfo
       begin(instance) / end(instance)  -> Iterator  (64-byte SBO, zero heap alloc)

MethodInfo
  Invoke(instance, args)  -> auto

MethodCreator<Func, func>  «non-type template param»
  owns 1 static MethodCall per function VALUE

── wiring ──────────────────────────────
TypeInfo  owns  PropertyInfo[], MethodInfo[]

02

Two Details Worth Knowing

Zero-allocation iteration.
A virtual-function iterator would new per traversal. Instead: a fixed 64-byte buffer, placement-new, lambda dispatch instead of a vtable, explicit deep-copy so Debug never double-frees a shallow buffer copy.
Static method objects without new.
MethodCreator<Func, func> keys on the function pointer value — each function gets its own template instantiation, and its own unique static MethodCall, for free.
