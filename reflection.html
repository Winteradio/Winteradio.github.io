C++17 Reflection

Three macros — GENERATE, PROPERTY, METHOD — modeled after Unreal Engine's UCLASS / UPROPERTY / UFUNCTION. They register a class's type, member variables, and member functions at compile time via C++17 inline static members, so the definitions live entirely in the header with no linker errors.

Type / Property / Method Registration

single-inheritance only, and that's what makes Cast<T> safe

TypeInfo currently models a single-inheritance chain — one SuperType per class, not a set of bases. Cast<T> walks that chain to do up/downcasting without dynamic_cast, which is exactly what single inheritance makes tractable: there's one unambiguous path from any type up to any of its ancestors, so a cast is just a walk, not a search across a DAG.

02

Two Details Worth Knowing About

Zero-allocation type erasure for container iteration.
A virtual-function iterator (IIterator*) would new on every traversal. Instead the iterator carries a fixed 64-byte buffer (alignas(void*) uint8_t[64]), placement-news the real STL iterator into it, and dispatches through captured lambdas instead of a vtable — with explicit deep-copy/destroy functions so Debug mode never sees a double-free from a shallow buffer copy.
Static method objects without new.
Two member functions with identical signatures would collide on the same static storage if keyed only by type. MethodCreator<Func, func> keys on the function pointer value as a non-type template parameter, so each function gets its own template instantiation — and its own unique static MethodCall — for free.
