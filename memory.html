02

No Compaction, By Design

Why compaction is out: Accessor lives inline in each instance's header, not a separate handle table. Moving an object during compaction moves its Accessor too — every live ObjectPtr/RootPtr already pointing at it would need finding and rewriting. Without a tracked referrer list, that's a second full GC pass on its own.
Fragmentation handled on allocation instead: each PoolBudget classifies buckets by occupancy (eSparse / eModerate / eDense / eFull) and routes new allocations to the lowest-occupancy bucket first — keeps any one bucket from hitting eFull and forcing a fresh backing allocation.
